# Backend: Multi-Modal Vision-Language Assistant

FastAPI service that accepts an image plus natural language question and returns an answer generated by a vision-language model (currently a lightweight stub).

## Quickstart

```bash
cd backend
python -m venv .venv
. .venv/Scripts/activate  # on Windows; use .venv/bin/activate on Unix
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Open the interactive docs at `http://127.0.0.1:8000/docs`.

### Endpoints

- `GET /health` - readiness info (model + device)
- `POST /api/ask` - multipart form-data: `image` (PNG/JPEG), `question`, optional `mode`, optional `session_id`

## Configuration

Environment variables:

- `VLM_API_BASE_URL` - full URL of the remote VLM endpoint (required when `MODEL_BACKEND=remote`)
- `VLM_API_KEY` - API key for the VLM provider (optional)
- `VLM_MODEL_NAME` - remote model identifier
- `MODEL_BACKEND` - `remote` (default) or `stub`
- `LOG_LEVEL` - logging level (default: INFO)
- `ENVIRONMENT` - environment label
- `MAX_IMAGE_MB` - maximum upload size in megabytes (default: 5)
- `REQUEST_TIMEOUT` - HTTP timeout for model calls (seconds)

See `.env.example` for a template.

## Tests

```bash
cd backend
pytest
```

Tests cover `/health`, `/api/ask` happy paths and validation, session store behaviour, config loading, and the VLM service stub/remote error handling.

## Docker

Build and run with Docker:

```bash
docker build -t mmv-backend .
docker run -p 8000:8000 --env-file .env mmv-backend
```
